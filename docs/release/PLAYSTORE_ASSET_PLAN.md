# Play Store Asset Plan

## Goal

Turn the current Guarden visuals and polished Android surfaces into a clean Play Store listing set.

## Provided Visuals

1. Wide branded lock banner
2. Shield-style Guarden icon
3. White-background unlocked lock render

## Recommended Asset Mapping

### App Icon

Use:

- Shield-style Guarden icon

Reason:

- Strongest brand recognition
- Matches in-app identity
- Reads clearly at small sizes

### Feature Graphic

Use:

- Wide branded lock banner

Adjustments:

- Remove store badges if possible
- Shorten title from `Guarden Secure Password Safe` to `Secure Password Vault` or `Guarden`
- Keep left-side copy large and simple
- Prefer a badge-free export for the final Play upload

### Screenshot Set

Suggested order:

1. Welcome screen with offline-first pill and launch chips
2. Login screen with the biometric unlock card
3. Home shell with Guarden header and offline-first sublabel
4. Bank Accounts empty state with the add CTA visible
5. Web Passwords empty state with the add CTA visible
6. Settings screen with the launch trust card plus backup / biometric controls

Cover card option:

- Use the white-background unlocked lock render for website or press materials, not as a replacement for real Play screenshots

## Screenshot Style Rules

- Use a consistent dark Guarden frame/background
- Put one short benefit line per screenshot
- Avoid paragraphs on screenshots
- Highlight one feature per image
- Keep typography large enough for small phone previews
- Keep the primary CTA visible when capturing empty-state modules
- Capture real app surfaces without ad hoc graphic overlays beyond the short headline

## Suggested Screenshot Headlines

1. `Offline-first encrypted vault`
2. `Unlock fast with biometrics`
3. `Guarden keeps your vault clear and local`
4. `Add bank and finance records fast`
5. `Keep web credentials organized`
6. `Review backup and privacy controls`

## Operator Capture Notes

- Capture on an Android phone-sized viewport with the dark Guarden theme active.
- Use a realistic but tidy data state for the home shell screenshot.
- Use truly empty module states for Bank Accounts and Web Passwords so the CTA button is visible.
- Avoid screenshots where the keyboard, snackbars, or permission popups cover the primary content.
- Keep the settings screenshot centered on the launch trust card and the Drive backup section.

## Missing Assets Still Needed

- Real Android phone screenshots from the polished surfaces above
- Optional 7-inch tablet screenshots
- Clean feature graphic export without store badges
- Optional promotional website / support page for linking
