**Step 2: Create AuthService** (lib/services/auth_service.dart)

Create an `AuthState` enum: `initial`, `unauthenticated`, `authenticated`, `locked`, `setupRequired`.

Create `authStateProvider` as `StateProvider<AuthState>((ref) => AuthState.initial)`.

Create `cryptoServiceProvider` as `Provider<CryptoService>((ref) => CryptoService())`.

Create `AuthService` class that takes `Ref` in constructor and accesses `secureStorageProvider` and `cryptoServiceProvider` via ref.

Create `authServiceProvider` as `Provider<AuthService>((ref) => AuthService(ref))`.

AuthService methods:

1. `Future<void> checkAuthState()` - reads `isSetupComplete` from storage. If not complete, sets authState to `setupRequired`. If complete but not authenticated, sets to `unauthenticated`.

2. `Future<SetupResult> setupMasterPassword(String password, String confirmPassword)` - validates passwords match and length >= 8. Generates a random salt (32 char hex string using Random.secure). Derives key via CryptoService.deriveKey(password, salt). Converts derived key bytes to base64 string (this IS the password hash). Generates seed phrase via CryptoService.generateSeedPhrase(). Encrypts seed phrase with the derived key via CryptoService.encryptText(). Saves to SecureStorageService: masterPasswordHash (base64 of derived key), salt, encryptedSeedPhrase, setupComplete. Sets authState to `authenticated`. Returns `SetupResult(success: true, seedPhrase: seedPhrase)`.

3. `Future<bool> verifyMasterPassword(String password)` - reads salt and stored hash from storage. Derives key from password + salt. Compares base64 of derived key bytes with stored hash. If match: sets authState to `authenticated`, returns true. If no match: returns false.

4. `Future<bool> recoverWithSeedPhrase(String seedPhrase, String newPassword)` - reads stored encrypted seed phrase and salt. Derives OLD key from... wait, we don't have the old password. Recovery approach: Store the seed phrase hash separately for verification. Actually simpler: store the seed phrase itself encrypted with a key derived FROM the seed phrase. So during setup: derive a recovery key from seedPhrase as password with a fixed known salt (e.g., 'guarden_recovery_salt'). Encrypt the seed phrase with THAT key too. Store separately as `_seedPhraseVerificationKey`. During recovery: user enters seed phrase -> derive recovery key from it -> decrypt the stored encrypted seed phrase -> if matches input, seed phrase is valid -> now re-setup with new password (new salt, new derived key, re-encrypt seed phrase with new key, save all).

Simplified recovery flow:
- `recoverWithSeedPhrase(seedPhrase, newPassword, confirmPassword)`:
  - Derive recovery key from seedPhrase using fixed salt 'guarden_recovery'
  - Try to decrypt stored `_seedPhraseEncryptedKey` with recovery key
  - If decryption succeeds AND decrypted text == seedPhrase, it's valid
  - Call internal _resetWithNewPassword(newPassword, seedPhrase) which does same as setupMasterPassword but reuses the validated seedPhrase
  - Returns true on success, false on invalid seed phrase

5. `void lock()` - sets authState to `locked`.

6. `void logout()` - sets authState to `unauthenticated`.

Create `SetupResult` class: `final bool success; final String? seedPhrase; final String? error;`

IMPORTANT: During `setupMasterPassword`, encrypt the seed phrase with BOTH:
- The master password derived key (stored as `_seedPhraseEncryptedKey`) -- this is for recovery verification
- Actually, use the recovery approach: derive key from seed phrase itself with fixed salt 'guarden_recovery', encrypt seed phrase with that key. This way during recovery, user provides seed phrase -> we derive the same key -> decrypt -> verify it matches.

So the storage layout is:
- `_masterPasswordHashKey`: base64(PBKDF2(password, random_salt))
- `_saltKey`: the random salt
- `_seedPhraseEncryptedKey`: AES-GCM-encrypt(seedPhrase, key=PBKDF2(seedPhrase, 'guarden_recovery'))
- `_isSetupCompleteKey`: 'true'
