    1. **SettingsService/Provider** — Add autofill category preferences:
       - `_autofillWebPasswordsKey = 'autofillWebPasswords'` (default: true)
       - `_autofillBankAccountsKey = 'autofillBankAccounts'` (default: false)
       - `_autofillSubscriptionsKey = 'autofillSubscriptions'` (default: false)
       - Getter/setter pairs in SettingsService. State fields + toggle methods in SettingsState/SettingsNotifier.

    2. **AutofillService** — Create `lib/services/autofill_service.dart`:
       - Uses `flutter_autofill_service` package (already in pubspec).
       - Import `package:flutter_autofill_service/flutter_autofill_service.dart`.
       - **Key methods:**
         a. `hasAutofillServicesSupport()` — Check if device supports autofill.
         b. `requestSetAutofillService()` — Prompt user to enable Guarden as autofill provider in system settings.
         c. `isAutofillServiceEnabled()` — Check if already enabled.
         d. `getMatchingCredentials(String? domain, String? packageName)` — Domain matching:
            - Normalize domain (strip www., protocol).
            - Search WebPassword list where `url` contains domain.
            - If bank autofill enabled, also search bank accounts.
            - Return list of `AutofillCredential` (custom class with id, title, username, domain).
         e. `provideCredential(String credentialId)` — Decrypt and return username+password.
            - MUST call `biometricService.authenticate()` first.
            - If biometric fails, return null (deny autofill).
            - If success, decrypt password using CryptoService + SecureStorage key.
         f. `setAutofillResponse(String username, String password)` — Use flutter_autofill_service to respond.
       - **AutofillCredential** data class: `{String id, String title, String username, String domain}`.
       - Riverpod provider: `autofillServiceProvider`.

    3. **Domain matching strategy** (Claude's discretion per CONTEXT.md):
       - Extract domain from URL: `Uri.parse(url).host` → strip 'www.' prefix.
       - Match by checking if stored domain contains request domain or vice versa.
       - Also match by app package name if available (store packageName in WebPassword.url or match against title).
       - Simple substring matching is acceptable for MVP.
  </action>
