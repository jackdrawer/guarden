# Plan 07-04: Telemetry Integration (Sentry & Firebase Analytics)

## Goal
Integrate Crashlytics/Crash Reporting (using Sentry as it's privacy-friendly and powerful) and Firebase Analytics for tracking app usage and freemium metrics. Ensure strict PII (Personally Identifiable Information) scrubbing so zero sensitive data (passwords, emails, vault keys) ever leaves the device.

## Rationale
To be competitive with Google and Bitwarden on the Play Store, we must have visibility into app stability (crash rates) and user engagement (premium conversion rates, feature usage).

## Scope of Work

1. **Dependency Injection:**
   - Add `sentry_flutter`, `firebase_core`, `firebase_analytics`.

2. **Telemetry Service (`telemetry_service.dart`):**
   - Create a central wrapper for both Sentry and Firebase.
   - Implement `logEvent(String name, {Map<String, dynamic>? parameters})` methods.
   - Implement `recordException(dynamic exception, StackTrace? stack)` method.

3. **PII Scrubbing Mechanisms:**
   - For Sentry: Add a `beforeSend` callback in `SentryFlutter.init` that explicitly strips out any strings that look like emails, passwords, or keys.
   - For Firebase Analytics: Only track predefined enum events (e.g., `premium_purchase`, `login_success`, `item_added`).

4. **App Initialization:**
   - Update `main.dart` to initialize these services safely.
   - Inject `TelemetryService` inside `ErrorHandler.handleGlobalError` to auto-log non-fatal AppErrors into Sentry cleanly.

## Execution Steps
1. Add dependencies to `pubspec.yaml`
2. Create `telemetry_service.dart` and `telemetry_provider.dart`
3. Modify `main.dart` error catchers.
4. Modify `ErrorHandler` to report strictly non-PII errors to telemetry.
