<task type="auto">
  <name>Task 2: Create MonitoringService with Sentry integration and PII scrubbing</name>
  <files>lib/services/monitoring_service.dart</files>
  <action>
Create comprehensive monitoring service in `lib/services/monitoring_service.dart`:

**MonitoringService class:**

1. `static Future<void> init()` - Initialize Sentry
   - Load SENTRY_DSN from dotenv (use dotenv package)
   - Call Sentry.init() with configuration:
     - dsn: from env
     - environment: kDebugMode ? 'development' : 'production'
     - tracesSampleRate: 0.2 (20% of transactions for performance monitoring)
     - beforeSend: _scrubPII callback
     - integrations: [FlutterErrorIntegration()]
   - If DSN is empty/missing, log warning and skip Sentry init (graceful degradation)

2. `static SentryEvent? _scrubPII(SentryEvent event, Hint hint)`
   - Filter event data recursively
   - Remove any field where key contains: 'password', 'seed', 'key', 'pin', 'secret', 'token', 'cipher'
   - Remove any field where value matches patterns: starts with 'sk_', starts with 'pk_' (API keys)
   - Preserve stack traces and error types
   - Return scrubbed event

3. `static Future<void> captureError(dynamic error, StackTrace? stackTrace, {Map<String, dynamic>? extras})`
   - Wrapper around Sentry.captureException
   - If error is AppError, add both technical and user messages as context
   - Add extras as breadcrumbs
   - Ensure no PII in extras

4. `static void addBreadcrumb(String message, {String? category, Map<String, dynamic>? data})`
   - Wrapper around Sentry.addBreadcrumb
   - Scrub data parameter before adding
   - Common categories: 'navigation', 'user.action', 'state.change'

**Provider:**
```dart
final monitoringProvider = Provider<MonitoringService>((ref) => MonitoringService());
```

**Error integration:**
- Import lib/errors/app_errors.dart
- When capturing AppError, add custom scope with:
  - tag 'error_type': error.runtimeType.toString()
  - context 'user_message': error.userMessage
  - context 'can_retry': error.canRetry

**Why aggressive PII filtering:** Per CONTEXT.md and FR3 requirement: "PII data is scrubbed from all crash reports and analytics." Password manager contains highly sensitive data - must never leak to monitoring systems.

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // Initialize monitoring (Sentry + Firebase)
  await dotenv.load(fileName: ".env"); // Load environment variables
  await MonitoringService.init(); // Initialize Sentry
  await AnalyticsService.init(); // Initialize Firebase

  // Existing initialization
  LocaleSettings.useDeviceLocale();

  // Wrap runApp with Sentry error boundary
  runZonedGuarded(() {
    runApp(ProviderScope(child: TranslationProvider(child: const GuardenApp())));
  }, (error, stackTrace) {
    MonitoringService.captureError(error, stackTrace);
  });
}
```
