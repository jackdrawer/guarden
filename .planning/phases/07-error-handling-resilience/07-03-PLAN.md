  <action>
Migrate first 3 providers from `AutoDisposeNotifier<T>` to `AsyncNotifier<T>`:

**Pattern for each provider:**

1. Change class signature:
   - FROM: `class XNotifier extends AutoDisposeNotifier<T>`
   - TO: `class XNotifier extends AsyncNotifier<T>`

2. Update build() method:
   - FROM: `T build() { return initialValue; }`
   - TO: `Future<T> build() async { return initialValue; }`

3. Wrap state mutations in try-catch:
   - FROM: `state = newValue;`
   - TO: `state = AsyncValue.data(newValue);` on success
   - TO: `state = AsyncValue.error(error, stackTrace);` on caught exceptions

4. Update provider declarations:
   - FROM: `NotifierProvider.autoDispose<XNotifier, T>`
   - TO: `AsyncNotifierProvider.autoDispose<XNotifier, T>`

**auth_provider.dart:**
- Migrate AuthNotifier to AsyncNotifier<AuthState>
- Wrap setupVault, login, verifyMasterPassword, recoverWithSeedPhrase in try-catch
- Catch StorageError, CryptoError, DatabaseError and set state to AsyncValue.error
- Existing `catch (_)` blocks on lines 32, 88, 130, 142 → replace with typed error handling
- Return AsyncValue.data(AuthState.authenticated) on success

**bank_account_provider.dart:**
- Migrate BankAccountNotifier to AsyncNotifier<List<BankAccount>>
- Wrap addBankAccount, updateBankAccount, deleteBankAccount in try-catch
- Catch DatabaseError from Hive operations
- Set state to AsyncValue.error on failures

**subscription_provider.dart:**
- Same pattern as bank_account_provider

**Why AsyncNotifier:** Per CONTEXT.md decision: "Migrate all providers from synchronous Notifier<T> to AsyncNotifier<T> returning AsyncValue<T>. UI uses .when(data:, error:, loading:) pattern for error states."
